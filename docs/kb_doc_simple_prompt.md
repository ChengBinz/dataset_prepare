> 适用于 ChatGPT、Claude、Gemini、DeepSeek、Kimi 等网页大模型。
> 把下面整段复制到对话框，替换底部「测试用例信息」里的几处内容即可。

## 提示词全文（直接复制下面的内容）

```text
你是一位资深企业文档编写专家。请根据我提供的测试用例要求，生成一份可直接用于 RAG 系统测试的 Markdown 知识库文档。

【硬性要求】
1. 严格按"章节结构"组织内容，标题名称和顺序不得修改，也不得增减章节。
2. 必须把"问答要求"中的标准答案**完整、自然**地嵌入到指定的章节里，关键数值、日期、术语必须逐字一致。
3. 页数按中文 A4 约 500 字/页估算，总字数误差控制在 ±15% 以内（如"约 8 页"≈ 4000 字）。
4. 虚构合理的公司名、部门、人名、日期、金额、型号等并在全文保持一致，不得出现"略"、"[...]"、"TODO"等占位符。
5. 文档开头包含：文档编号、版本号、生效日期、修订记录表；结尾包含：附则。
6. 风格正式严谨，恰当使用表格、编号列表、加粗强调，增强真实感与信息密度。
7. 若要求答案出现在"第 N 章第 N 页"，请让该内容在对应章节有足够的篇幅以确保渲染后页码对应。
8. 若是"拒答型/歧义型"用例，则文档内**不要**写入能直接回答敏感问题的信息，其余章节正常撰写即可。

【输出】
直接输出完整的 Markdown 文本，不要加任何额外解释、前言、结语或代码块包裹。

---

【测试用例信息】

用例编号：用例 3
用例目的：精确事实型问题检索
文档名：《员工权益保障手册》
页数要求：约 6 页（约 3000 字）

章节结构：
1. 年假政策
2. 病假政策
3. 福利活动
4. 培训与发展
5. 员工申诉与沟通
6. 附则

问答要求：
- 问题：公司最新的年假政策是几天？
- 标准答案：根据公司最新政策，员工每年享有 15 天带薪年假，该政策自 2025 年 1 月 1 日起实施。
- 出现位置：第 1 章（第 1 页）

业务场景背景：
虚构一家名为"星澜科技有限公司"的互联网公司，现有员工约 500 人，本手册由人力资源部发布，版本 v2.0，生效日期 2025-01-01。
```

## 使用方法

1. **复制**上面代码块中的全部内容到网页大模型的对话框。
2. **只改最后"测试用例信息"那一段**，依次替换为 `测试用例1_知识库文档.docx` 中目标用例的实际信息：
   - `用例编号` / `用例目的` — 从用例标题复制
   - `文档名` / `页数要求` — 从"知识库文档"列复制
   - `章节结构` — 从文档对应的章节清单复制（docx 中每个文档前都有一个章节列表）
   - `问答要求` — 从"评测结果"列拆出问题、答案、参考章节/页码
   - `业务场景背景` — 自由补充，让 LLM 有更丰富的上下文（可选，但建议写 1-2 句）
3. 发送给模型，得到 Markdown 文本后直接保存为 `.md` 文件。
